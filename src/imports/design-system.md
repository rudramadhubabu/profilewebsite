Create the design system section for the Discount Tire mobile application UX case study.

The design system must follow the existing brand identity used on the Discount Tire website.

Do not invent a new color palette.

Maintain visual consistency with the brand’s existing digital experience.

Use a clean automotive retail interface style with bold hierarchy and strong contrast.

Balance text explanations with visual component examples.

---

DESIGN SYSTEM OVERVIEW

The mobile design system was built by adapting the existing Discount Tire brand language into a scalable mobile interface.

The goal was to maintain brand familiarity while improving usability for mobile users.

Key design principles:

Clarity
Speed
Trust
Automotive expertise
Service transparency

---

COLOR SYSTEM

Primary Brand Red

Use the same red tone used for website CTAs.

Usage:

Primary buttons
Promotions
Important highlights
Call-to-action elements

Neutral Colors

Black
Used for headers and navigation

Dark Gray
Used for secondary text and icons

Light Gray
Used for dividers and background sections

White

Primary application background

Maintain strong contrast between red CTAs and white surfaces.

---

TYPOGRAPHY SYSTEM

Use a bold modern sans-serif typeface consistent with the brand style.

Typography hierarchy:

Hero Title
Large bold headings used for marketing banners and key messages.

Section Headers
Used for category titles such as Tires, Wheels, Appointments.

Body Text
Used for product descriptions and service details.

Small UI Labels
Used for filters, tags, and vehicle specifications.

Typography should feel:

Bold
Clear
Highly readable
Optimized for mobile scanning.

---

ICONOGRAPHY

Use simple automotive-themed line icons.

Examples:

Car icon
Tire icon
Location icon
Calendar icon
Service tools icon

Icons should be minimal and consistent in stroke weight.

---

BUTTON SYSTEM

Primary Button

Red background
White text
Rounded corners

Used for actions such as:

Book Appointment
Add Vehicle
View Tire Details

Secondary Button

White background
Red border
Red text

Used for secondary actions.

Tertiary Button

Text only with red highlight.

---

CARD COMPONENTS

Cards are used heavily for product browsing.

Examples:

Tire product cards
Store location cards
Appointment cards
Vehicle profile cards

Card structure:

Product image
Product title
Price
Performance rating
CTA button

Cards should have subtle shadows and rounded corners.

---

PRODUCT TILE DESIGN

Each tire product tile should contain:

Tire image
Brand name
Tire model
Price
Customer rating
Performance attributes

Example attributes:

Dry traction
Wet traction
Snow performance
Tread life

Use small visual indicators for performance ratings.

---

NAVIGATION SYSTEM

Bottom navigation bar with five tabs.

Home
Shop Tires
My Vehicle
Appointments
Account

Navigation icons should be simple and easily recognizable.

---

INPUT COMPONENTS

Forms are required for vehicle entry and appointment booking.

Input field structure:

Label above field
Rounded input field
Clear validation messages

Example fields:

Vehicle year
Make
Model
Tire size
Store location

---

INTERACTION STATES

Define interactive states for all components.

Hover
Pressed
Loading
Disabled

Animations should be subtle and fast to maintain a responsive mobile experience.

---

DESIGN SYSTEM GOAL

The final design system provides a consistent and scalable foundation for the Discount Tire mobile application.

It ensures brand consistency while improving usability and mobile-first product discovery.
